
# Archival Document (Schema)

`ogc.heritage.archival-document` *v0.1*

A digital-representation profile constrained to archival document formats (PDF/A, LIDO/XML), with a mandatory persistent identifier for long-term archival reference.

[*Status*](http://www.opengis.net/def/status): Under development

## Description

## Archival Document

An `ArchivalDocument` is a [`digital-representation`](../digital-representation) constrained to
archival document formats — PDF/A or LIDO/XML — covering D8.2 profile C (long-term archival and
metadata-exchange records).

Unlike the other digital-representation profiles, `persistentIdentifier` is **mandatory** here:
archival records are expected to be citable and durably resolvable, which is the whole point of
migrating a legacy paper or filesystem archive into this model (UC-V-3).

## Examples

### A LIDO/XML catalogue record with a Handle PID
A LIDO/XML export of the fishing creel's catalogue record, migrated from a legacy archive (UC-V-3) and minted with a persistent Handle.
#### json
```json
{
  "id": "https://example.org/heritalise/digital/fishing-creel-lido-record",
  "identifier": "WHM-1923.45-LIDO",
  "isAbout": "https://example.org/heritalise/object/fishing-creel",
  "mediaType": "text/xml",
  "persistentIdentifier": "https://hdl.handle.net/10.1234/whm-1923.45"
}

```


## Schema

```yaml
$schema: https://json-schema.org/draft/2020-12/schema
title: Archival Document
description: "A `digital-representation` constrained to archival document formats
  \u2014 PDF/A or LIDO/XML \u2014 with\na mandatory persistent identifier, for long-term
  archival reference (D8.2 profile C).\n"
allOf:
- $ref: https://ogcincubator.github.io/bblocks-heritage/build/annotated/heritage/digital-representation/schema.yaml
- type: object
  required:
  - mediaType
  - persistentIdentifier
  properties:
    mediaType:
      type: string
      enum:
      - application/pdf
      - text/xml
      description: '`application/pdf` for a PDF/A document (ISO 19005), or `text/xml`
        for a LIDO/XML record. A PDF/A claim cannot be verified from the media type
        alone; validate the actual file separately if conformance must be checked.'

```
